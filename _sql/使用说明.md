# 检查和初始化扩展表使用说明

## 📋 数据库信息
- **地址**：127.0.0.1:3306
- **数据库**：free-fs
- **用户名**：root
- **密码**：582000

---

## 🚀 最简单的方法

### 方法一：直接执行检查和初始化脚本（推荐）

使用 Navicat、phpMyAdmin 或 MySQL 命令行执行这个文件：
```
D:\APPLICATION\free-fs\_sql\check_and_init.sql
```

这个脚本会自动：
1. ✓ 检查扩展表是否存在
2. ✓ 如果不存在就创建
3. ✓ 最后显示所有相关表

---

## 📝 其他方法

### 方法二：使用批处理文件检查

双击运行：
```
D:\APPLICATION\free-fs\_sql\check_tables.bat
```

### 方法三：在 MySQL 命令行手动检查

```bash
mysql -h 127.0.0.1 -P 3306 -u root -p582000
```

然后执行：
```sql
USE `free-fs`;
SHOW TABLES LIKE 'file_%';
```

### 方法四：使用 Navicat 等图形化工具

1. 打开 Navicat，连接到 MySQL
2. 选择 `free-fs` 数据库
3. 查看左侧表列表，搜索 `file_` 开头的表
4. 应该看到：
   - `file_audit_log`
   - `file_permission`
   - `file_version`

---

## ✅ 验证是否成功

执行查询后，您应该看到这三个表：
```
+---------------------------+
| Tables_in_free-fs (file_%) |
+---------------------------+
| file_audit_log            |
| file_permission           |
| file_version              |
+---------------------------+
```

---

## ❓ 如果表不存在怎么办？

执行以下任一脚本：

1. **完整初始化（推荐）**：`init_extension_tables.sql`
2. **仅更新现有表**：`update_file_audit_log_table.sql`
3. **智能检查并初始化**：`check_and_init.sql`
